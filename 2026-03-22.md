> 不考虑 env 并行，不考虑四足特异性。以固定基座 n-DOF 串联机械臂为例，
> 完整推导一步仿真 `step(q, qdot, tau) -> q_new, qdot_new` 的全部计算。
>
> **变换采用 SE(3) 约定**，与项目代码 (`spatial.py`) 一致。

## 0. 问题设定

**机械臂结构**：n 个刚体（link），编号 1..n，由 n 个旋转关节（revolute joint）串联。

```
World ──[joint 1]── body 1 ──[joint 2]── body 2 ── ... ──[joint n]── body n
(固定)
```

- `parent[i] = i - 1`（body 0 = world，固定不动）
- 所有关节均为 revolute，每个关节 1 个自由度
- 广义坐标 `q ∈ R^n`，广义速度 `qdot ∈ R^n`，关节力矩 `tau ∈ R^n`

**目标**：给定 `(q, qdot, tau)`，计算广义加速度 `qddot ∈ R^n`，然后积分得到 `(q_new, qdot_new)`。

## 1. SE(3) 变换约定

本项目的 `SpatialTransform(R, r)` 采用 SE(3) 约定：

### 1.1 存储量

```
R ∈ R^{3×3}   child→parent 旋转矩阵（R 将 child 坐标系的向量变换到 parent 坐标系）
r ∈ R^3        child 原点在 parent 坐标系中的位置
```

对应齐次矩阵（SE(3) 群元素）：

```
T = ┌      ┐
    │ R  r │ ∈ R^{4×4}
    │ 0  1 │
    └      ┘
```

含义：若点 p 在 child frame 中坐标为 p_c，则在 parent frame 中坐标为 `p_p = R @ p_c + r`。

### 1.2 空间速度（twist）

```
v = [v_lin; omega] ∈ R^6     (linear first, then angular — Pinocchio / Isaac Lab 约定)
```

- `v_lin ∈ R^3`：body frame 原点的线速度（在 body frame 中表达）
- `omega ∈ R^3`：角速度（在 body frame 中表达）

### 1.3 空间力（wrench）

```
f = [f_lin; tau] ∈ R^6
```

- `f_lin ∈ R^3`：力
- `tau ∈ R^3`：力矩（关于 body frame 原点）

对偶关系：功率 `P = f^T @ v` 在任何坐标系下不变。

### 1.4 速度变换：apply_velocity

将空间速度从 parent frame 变换到 child frame：

```python
# SpatialTransform(R, r).apply_velocity(v):
v_child_lin = R^T @ (v_parent_lin + omega_parent × r)
omega_child = R^T @ omega_parent
```

写成矩阵形式（令 `E = R^T`）：

```
         ┌              ┐
X_mot  = │  E    -E@rx  │ ∈ R^{6×6}       (rx = skew(r))
         │  0      E    │
         └              ┘

v_child = X_mot @ v_parent
```

**关键**：6×6 矩阵 `X_mot` 的构造用 `E = R^T`（parent→child 旋转），但底层**存储**的是 `R`（child→parent）。

### 1.5 力变换：apply_force

将空间力从 child frame 变换到 parent frame：

```python
# SpatialTransform(R, r).apply_force(f):
f_parent_lin = R @ f_child_lin
tau_parent   = R @ tau_child + r × (R @ f_child_lin)
```

矩阵形式：`f_parent = X_mot^T @ f_child`。

```
         ┌                ┐
X^T    = │  E^T       0   │ = │  R        0     │
         │  rx@E^T   E^T  │   │  rx@R     R     │
         └                ┘   └                  ┘
```

**验证对偶性**：`f_p^T @ v_p = (X^T @ f_c)^T @ (X^{-1} @ v_c) = f_c^T @ X @ X^{-1} @ v_c = f_c^T @ v_c` ✓

### 1.6 变换的组合与求逆

**组合** (先 other，后 self)：

```
(R_new, r_new) = (R_self @ R_other,  r_self + R_self @ r_other)
```

对应齐次矩阵乘法 `T_self @ T_other`。

**求逆**：

```
(R_inv, r_inv) = (R^T,  -(R^T @ r))
```

对应 `T^{-1} = [R^T, -R^T@r; 0, 1]`。

### 1.7 高效操作 vs 朴素 6×6

**核心思想**：6×6 空间变换矩阵 `X` 的 36 个元素全部由 `(R, r)` 共 12 个独立参数决定，
矩阵中存在大量结构性的零和重复。高效做法是**不展开成 6×6**，直接用 `(R, r)` 按解析公式计算，
避免冗余运算。在 ABA 等递归算法中每个关节都反复执行这些操作，累计节省非常可观。

| 操作 | 朴素 6×6 | 利用 (R,r) 结构 | 存储 |
|------|---------|----------------|------|
| `X @ v` | 6×6 matvec = 66 FLOP | 2 个 3×3 matvec + 1 cross = 36 FLOP | 12 floats vs 36 |
| `X^T @ f` | 同上 | 同上 36 FLOP | |
| `X1 @ X2` | 6×6 matmul = 396 FLOP | 1 个 3×3 matmul + 1 matvec = 60 FLOP | |
| `X^{-1}` | 6×6 求逆 | 转置 + 1 matvec = 18 FLOP | |

以 `X @ v`（速度变换，见 1.4）为例：直接用公式 `v_child_lin = R^T @ (v_parent_lin + omega × r)`、
`omega_child = R^T @ omega_parent`，只需 1 次叉积 + 2 次 3×3 matvec ≈ 36 FLOP。
而 `X⁻¹` 更直观（见 1.6）：转置 `R` 几乎免费，再做一次 3×3 matvec 即可，共 ≈ 18 FLOP。

## 2. 空间惯量 I ∈ R^{6×6}

### 2.1 定义

```
         ┌                        ┐
I_body = │  m·I_3      -m·skew(c) │ ∈ R^{6×6}
         │  m·skew(c)   I_rot     │
         └                        ┘
```

其中：
- `m`：质量（标量）
- `c ∈ R^3`：质心在 body frame 中的位置
- `skew(c) ∈ R^{3×3}`：反对称矩阵，`skew(c) @ x = c × x`
- `I_rot = I_CoM + m·(|c|²·I_3 - c@c^T) ∈ R^{3×3}`：关于 body 原点的转动惯量（平行轴定理）
- `I_CoM ∈ R^{3×3}`：关于质心的转动惯量

### 2.2 性质

| 性质 | 说明 |
|------|------|
| 对称 | `I = I^T`：右上角 `(-m·cx)^T = m·cx` = 左下角 ✓ |
| 正定 | `v^T @ I @ v = m|v_CoM|² + ω^T @ I_CoM @ ω > 0`（动能为正） |
| 独立元素 | 21 个（对称 6×6） |
| 特殊情况 | 若 `c = 0`（原点在质心），右上和左下块为零，退化为 `diag(m·I_3, I_CoM)` |

### 2.3 高效 I @ v

```
h = I @ v:
    h_lin = m · v_lin - m · (c × omega)       # scale + cross: 9 FLOP
    h_ang = m · (c × v_lin) + I_rot @ omega   # cross + matvec(3×3 对称): 6+15 = 21 FLOP
    总计: ~30 FLOP（vs 朴素 6×6 matvec 66 FLOP）
```

### 2.4 坐标变换（congruence transform）

将惯量从 child frame 变换到 parent frame：

```
I_parent = X^T @ I_child @ X
```

其中 `X` 是 parent→child 的 motion transform。这是一个 **congruence transform**（保持对称性和正定性）。

## 3. 空间叉积

### 3.1 运动叉积 crm（motion × motion → motion）

对 `v = [v_lin; omega]`：

```
              ┌                          ┐
crm(v)      = │ skew(omega)   skew(v_lin) │ ∈ R^{6×6}
              │     0         skew(omega) │
              └                          ┘
```

高效实现（不构造矩阵）：

```
crm(v) @ u:
    result_lin = omega × u_lin + v_lin × u_ang     # 2 cross = 12 FLOP
    result_ang = omega × u_ang                      # 1 cross = 6 FLOP
    总计: 18 FLOP
```

### 3.2 力叉积 crf（motion × force → force）

```
crf(v) = -crm(v)^T

              ┌                          ┐
crf(v)      = │ skew(omega)       0      │ ∈ R^{6×6}
              │ skew(v_lin)  skew(omega) │
              └                          ┘
```

高效实现：

```
crf(v) @ f:
    result_lin = omega × f_lin                      # 6 FLOP
    result_ang = v_lin × f_lin + omega × f_ang      # 12 FLOP
    总计: 18 FLOP
```

**用途**：偏置力 `pA_i = crf(v_i) @ (I_i @ v_i)` — Coriolis/离心力。

## 4. 关节运动子空间 S ∈ R^6

对 revolute joint（绕轴 k 旋转）：

```
S = [0(3); k] ∈ R^6     (线速度为 0，角速度沿关节轴 k)
```

其中 `k ∈ R^3` 是 body frame 中的单位向量。

关节速度贡献：`vJ = S · qdot_i ∈ R^6`（标量 × 6-vector）。

**S 的稀疏性**：若 k 是坐标轴（如 `k = [0,0,1]`），S 只有 1 个非零元素。则：
- `S^T @ x` = 取 x 的一个分量（0 FLOP）
- `S · q` = 在一个位置放 q，其余为 0（0 FLOP）

## 5. 固定数据（build 时确定，运行时只读）

对每个 body i = 1..n：

| 量 | 维度 | 说明 |
|----|------|------|
| `parent[i]` | int | = i-1（串联链） |
| `k_i` | R^3 | 关节旋转轴（body frame 中的单位向量） |
| `S_i = [0; k_i]` | R^6 | 运动子空间（稀疏） |
| `R_tree_i` | R^{3×3} | 树变换旋转（child→parent），正交 |
| `r_tree_i` | R^3 | 树变换平移（child 原点在 parent 坐标系中的位置） |
| `I_i` | R^{6×6} | 空间惯量（body frame 中，对称正定，21 独立元素） |

总存储（per body）：3 + 6 + 9 + 3 + 21 = 42 floats（利用对称性）。

## 6. 完整仿真一步

输入：`q ∈ R^n`, `qdot ∈ R^n`, `tau ∈ R^n`
输出：`q_new ∈ R^n`, `qdot_new ∈ R^n`

### Step 1: 关节变换

对每个 body i（可并行，无依赖）：

```
theta = q_i                                        # 关节角（标量）
c = cos(theta),  s = sin(theta)

# Rodrigues 公式：绕单位轴 k 旋转 theta
R_J = c·I_3 + (1-c)·k@k^T + s·skew(k)            ∈ R^{3×3}，正交，det=1

r_J = 0                                            # revolute 无平移
```

FLOP: ~45（1 cos + 1 sin + 外积 + scale + add）

若 k 是坐标轴，退化为 2×2 平面旋转嵌入 3×3，只需 4 个非零元素。

### Step 2: 复合变换

```
R_up_i = R_tree_i @ R_J_i                          # 3×3 matmul: 45 FLOP
r_up_i = r_tree_i                                  # revolute: r_J = 0
```

`(R_up_i, r_up_i)` 是从 body i frame 到 parent(i) frame 的 SE(3) 变换。

注意：由于 revolute joint `r_J = 0`，一般情况的 `r_up = r_tree + R_tree @ r_J` 简化为 `r_up = r_tree`（常量，不随 q 变化）。

### Step 3: 正向运动学 FK

逐 body 递推（root → leaf），计算世界位姿 `(R_w_i, r_w_i)`：

```
for i = 1, 2, ..., n:                              # 链式依赖，顺序执行
    p = parent[i]
    if p == 0:                                      # parent 是 world
        R_w_i = R_up_i                              # child→world
        r_w_i = r_up_i                              # child 原点在 world 中的位置
    else:
        R_w_i = R_w_p @ R_up_i                      # SE(3) 组合: 45 FLOP
        r_w_i = r_w_p + R_w_p @ r_up_i              # 位置递推: 18 FLOP
```

等等——这里有一个微妙之处。`(R_up, r_up)` 是 child→parent 变换，链式组合后
`(R_w, r_w)` 是 child→world 变换。

验证：`p_world = R_w_i @ p_body + r_w_i`
= `R_w_p @ (R_up_i @ p_body + r_up_i) + r_w_p`
= `R_w_p @ R_up_i @ p_body + R_w_p @ r_up_i + r_w_p` ✓

FLOP per body: ~63
总 FLOP: ~63n

**FK 输出**：每个 body 的世界位姿 `(R_w_i, r_w_i)`，用于渲染和接触力计算。

### Step 4: ABA Pass 1 — 前向递推（root → leaf）

计算空间速度、偏置加速度，初始化关节空间惯量和偏置力。

```
for i = 1, 2, ..., n:                              # 链式依赖，顺序执行
    p = parent[i]

    # ---- 4a. 速度传播 ----
    vJ_i = S_i · qdot_i                             # R^6 = 标量 × 6-vec, 6 FLOP

    if p == 0:
        v_i = vJ_i                                  # parent velocity = 0 (fixed base)
    else:
        # apply_velocity: 将 parent 速度变换到 child frame，加上关节速度
        # v_child_lin = R_up^T @ (v_p_lin + omega_p × r_up)
        # omega_child = R_up^T @ omega_p
        v_i = apply_velocity(R_up_i, r_up_i, v_p) + vJ_i    # 36 + 6 = 42 FLOP

    # ---- 4b. 偏置加速度（Coriolis）----
    c_i = crm(v_i) @ vJ_i                           # 18 FLOP
    # 物理含义：v × vJ，非线性速度耦合项

    # ---- 4c. 初始化关节空间惯量 ----
    IA_i = copy(I_i)                                 # 拷贝 6×6 对称矩阵

    # ---- 4d. 初始化偏置力 ----
    Iv = I_i @ v_i                                   # 30 FLOP（利用惯量结构）
    pA_i = crf(v_i) @ Iv                             # 18 FLOP
    # pA = v ×_f (I·v) — Coriolis + 离心力

    # ---- 4e. 外力合入 ----
    pA_i -= f_ext_i                                  # 6 FLOP（接触力等，body frame）

FLOP per body: 6 + 42 + 18 + 30 + 18 + 6 = ~120
总 FLOP: ~120n
```

**中间结果**：

| 量 | 维度 | 性质 |
|----|------|------|
| `v_i` | R^6 | body i 的空间速度（body frame） |
| `c_i` | R^6 | 偏置加速度 |
| `IA_i` | R^{6×6} | 初始 = 刚体惯量，**对称正定**，后续 Pass 2 会被 children 累加修改 |
| `pA_i` | R^6 | 偏置力，后续 Pass 2 会被 children 累加修改 |

### Step 5: 接触力计算（可选，在 Pass 1 和 Pass 2 之间）

利用 FK 结果 `(R_w, r_w)` 和 body 速度 `v` 计算接触力，合入 `f_ext`。

```
for each contact point c:
    body = c.body_id
    pos_local = c.position                              # R^3, body frame

    # 世界位置
    pos_world = R_w[body] @ pos_local + r_w[body]       # 15 FLOP

    # 穿透检测
    depth = -pos_world[2]                               # 地面在 z=0
    if depth <= 0: continue

    # 世界速度
    omega_body = v[body][3:]                            # angular part
    v_lin_body = v[body][:3]                            # linear part
    omega_world = R_w[body] @ omega_body                # 15 FLOP
    v_world = R_w[body] @ (v_lin_body + omega_body × pos_local)   # 21 FLOP

    # Penalty 接触力（世界坐标系）
    f_normal = max(0, k_n · depth - b_n · v_z)          # 弹簧-阻尼
    v_tangent = [v_x, v_y, 0]
    f_friction = -mu · f_normal · v_tangent / sqrt(|v_t|² + eps²)
    f_world = [f_friction_x, f_friction_y, f_normal]

    # 变换到 body frame，合入 f_ext
    f_body_lin = R_w[body]^T @ f_world                  # 15 FLOP
    tau_body = pos_local × f_body_lin                    # 6 FLOP
    f_ext[body] += [f_body_lin; tau_body]                # 6 add
```

### Step 6: ABA Pass 2 — 反向递推（leaf → root）

**这是计算量最大的环节。** 将关节空间惯量和偏置力从 child 传播到 parent。

```
for i = n, n-1, ..., 1:                             # 逆序，链式依赖
    p = parent[i]

    # ---- 6a. 计算中间量 ----
    U_i = IA_i @ S_i              # IA 的后 3 列 × k: ~30 FLOP → R^6
    d_i = S_i^T @ U_i             # = k^T @ U[3:6]: ~5 FLOP → 标量 (> 0)
    d_inv_i = 1 / d_i             # 1 FLOP

    u_i = tau_i - S_i^T @ pA_i    # = tau - k^T @ pA[3:6]: ~6 FLOP → 标量
```

**U = IA @ S 的结构**：

由于 `S = [0; k]`，`IA @ S` 只需取 IA 的右 3 列做 matvec：

```
U = IA @ [0; k] = [IA[0:3, 3:6] @ k;  IA[3:6, 3:6] @ k]
```

即两个 3×3 matvec = 30 FLOP。若 k 是坐标轴，退化为取 IA 的一列 = 0 FLOP。

**d 的物理含义**：关节空间的有效惯量。因为 IA 正定且 S ≠ 0，所以 `d = S^T @ IA @ S > 0`。

```
    # ---- 6b. 向 parent 传播的修正惯量和力 ----
    IA_A_i = IA_i - d_inv_i · U_i @ U_i^T     # rank-1 update
    pA_A_i = pA_i + IA_A_i @ c_i + U_i · (d_inv_i · u_i)
```

**IA_A 的性质**：
- `IA_A = IA - (1/d)·U@U^T`：rank-1 降秩
- 对称（IA 对称，`U@U^T` 对称）
- **半正定**（从正定矩阵减去投影，秩降为 5）
- 物理含义：body i "去掉关节自由度后"传递给 parent 的等效惯量

rank-1 update FLOP（利用对称性只算上三角 21 元素）：21 mul + 21 sub = 42 FLOP

```
    # ---- 6c. 向 parent 累加 ----
    if p != 0:
        # 惯量传播: congruence transform
        IA_p += X_up_i^T @ IA_A_i @ X_up_i       # 见下文详解

        # 力传播: apply_force (child → parent)
        pA_p += apply_force(R_up_i, r_up_i, pA_A_i)   # 36 FLOP
```

### Congruence Transform 详解

`X^T @ M @ X`，其中 `X` 用 `(E, rx)` 表示（`E = R^T`，`rx = skew(r)`）：

```
     ┌              ┐       ┌            ┐       ┌              ┐
X  = │  E    -E@rx  │,  M = │ M11   M12  │,  X^T=│  E^T     0   │
     │  0      E    │       │ M12^T M22  │       │ -rx@E^T  E^T │
     └              ┘       └            ┘       └              ┘
```

其中 M11, M22 ∈ R^{3×3} 对称, M12 ∈ R^{3×3} 一般。

改写为 SE(3) 存储量 `(R, r)`，其中 `E = R^T`：

```
先算 M @ X（右乘）:
  (MX)_11 = M11 @ R^T                               # 3×3 matmul: 45 FLOP
  (MX)_12 = -M11 @ R^T @ rx + M12 @ R^T             # 2 个 matmul + add: 93 FLOP
  (MX)_21 = M12^T @ R^T                             # 45 FLOP
  (MX)_22 = -M12^T @ R^T @ rx + M22 @ R^T           # 93 FLOP

再算 X^T @ (MX)（左乘）:
  Result_11 = R @ (MX)_11                            # 45 FLOP → 对称
  Result_12 = R @ (MX)_12                            # 45 FLOP
  Result_21 = -rx @ R @ (MX)_11 + R @ (MX)_21       # 2 × 45 + add: 93 FLOP
  Result_22 = -rx @ R @ (MX)_12 + R @ (MX)_22       # 93 FLOP → 对称
```

注意：`Result_21 = Result_12^T`（整体矩阵对称），所以可以跳过 Result_21 的计算。

```
总 FLOP（朴素分块）: 45×8 + add ≈ 400 FLOP
优化（跳过 Result_21，利用对称性）: ~320 FLOP
vs 朴素两次 6×6 matmul: 2 × 396 = 792 FLOP
```

**进一步优化**：注意到很多中间量可复用：

```
P = R @ M11 @ R^T        # 对称结果，但中间过程是 2 个 3×3 matmul
Q = R @ M12 @ R^T        # 一般 3×3
S = R @ M22 @ R^T        # 对称结果

Result_11 = P
Result_12 = -P @ rx + Q
Result_22 = rx^T @ P @ rx - rx^T @ Q - Q^T @ rx + S
          = rx @ P @ rx + rx @ Q + Q^T @ rx + S       (因为 rx^T = -rx)
```

这种形式用 3 个 "旋转相似变换" `R @ M_block @ R^T` + 若干 rx 乘法。

**Pass 2 per-body FLOP 总结**：

| 操作 | FLOP |
|------|------|
| U = IA @ S | 30 |
| d = S^T @ U | 5 |
| u = tau - S^T @ pA | 6 |
| IA_A = IA - UU^T/d | 42 |
| pA_A = pA + IA_A@c + U·(u/d) | ~60 |
| **congruence: X^T @ IA_A @ X** | **~350** |
| force transform: apply_force(pA_A) | 36 |
| **总计** | **~530** |

**总 FLOP（Pass 2）: ~530n**

### Step 7: ABA Pass 3 — 前向递推（root → leaf）

```
for i = 1, 2, ..., n:                               # 链式依赖，顺序执行
    p = parent[i]

    # ---- 7a. Parent 加速度变换到 child frame ----
    if p == 0:
        # 重力处理: a_0 = -gravity (Featherstone 约定)
        # 将 -gravity 从 world frame 变换到 body 1 frame
        a_from_parent = apply_velocity(R_up_i, r_up_i, -gravity)     # 36 FLOP
    else:
        a_from_parent = apply_velocity(R_up_i, r_up_i, a_p)          # 36 FLOP

    a_prime = a_from_parent + c_i                    # 6 add

    # ---- 7b. 关节加速度 ----
    qddot_i = d_inv_i · (u_i - U_i^T @ a_prime)
    # = (1/d) · (u - S^T @ IA @ a')... 但 U = IA @ S 已在 Pass 2 算好
    # 实际: scalar = scalar · (scalar - dot(R^6, R^6))
    # FLOP: dot(6) + sub + mul = 13 FLOP

    # ---- 7c. Body 加速度 ----
    a_i = a_prime + S_i · qddot_i                    # 12 FLOP

FLOP per body: 36 + 6 + 13 + 12 = ~67
总 FLOP: ~67n
```

**输出**：`qddot ∈ R^n`（每个关节的角加速度）。

### Step 8: 积分（半隐式欧拉）

```
# 先更新速度（用新加速度）
qdot_new = qdot + dt · qddot              # n mul + n add = 2n FLOP

# 再更新位置（用新速度）— 半隐式关键
q_new = q + dt · qdot_new                 # n mul + n add = 2n FLOP

FLOP: 4n
```

对 revolute joint，q 和 qdot 都是标量，积分就是标量运算。

## 10. 串联链的并行性

对于串联链 parent[i] = i-1：

```
Pass 1 (forward):  body 1 → 2 → 3 → ... → n       完全顺序，0 并行
Pass 2 (backward): body n → n-1 → ... → 1          完全顺序，0 并行
Pass 3 (forward):  body 1 → 2 → 3 → ... → n       完全顺序，0 并行
```

总共 3n 次顺序操作。**单环境内无法并行（串联链固有限制）。**

可并行的只有：
- Step 1-2（关节变换）：n 个 body 独立
- Step 5（接触力）：各接触点独立
- Step 8（积分）：n 个关节独立

但这些环节计算量占比很小。

对于**树结构**（如四足），同层 sibling 可以并行——见 2026-03-21.md 讨论。

## 11. CRBA 替代方案简述

```
CRBA (Composite Rigid Body Algorithm):
  1. 递推计算质量矩阵 M ∈ R^{n×n}     O(n²)
  2. RNEA 计算偏置力 C ∈ R^n            O(n)
  3. Cholesky 分解 M = L@L^T           O(n³/3)
  4. 前后代换 L@L^T@qddot = tau-C      O(n²)
```

| | ABA | CRBA |
|---|-----|------|
| 复杂度 | O(n) | O(n²) + O(n³) |
| n=7 FLOP | ~6100 | ~5500 |
| n=24 FLOP | ~18000 | ~64000 |
| M 矩阵 | 不产生 | 显式 n×n，可走 Tensor Core |
| 串联链并行性 | 无 | Cholesky 有一定并行性 |
| GPU 友好性 | 小矩阵链式依赖 | n×n 矩阵运算，规模更大 |

**n ≤ 10 时两者计算量相当；n > 15 后 ABA 显著更优。**

但在 GPU 上，CRBA 产生的 n×n 矩阵更适合 batched GEMM / Cholesky，这是一个值得考虑的 trade-off（见 2026-03-21.md Q3）。
